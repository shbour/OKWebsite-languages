 How it works
OK basics for new users

 Comment cela fonctionne
Principes de base de OK pour les nouveaux utilisateurs

Une fois que vous avez installé un portefeuille OKCash sur votre ordinateur, votre téléphone mobile ou votre périphérique IoT, il génère votre première adresse OK et vous pouvez créer plus d'adresses lorsque vous en avez besoin. C'est très similaire à la façon dont fonctionne le courrier électronique, sauf que les adresses OKCash ne devrait être utilisé qu'une seule fois.

Balances - OK block chain

Balances - chaîne de blocs OK

The OK block chain is a shared public ledger on which the entire OKCash network relies. All confirmed transactions are included in the OK block chain. This way, OKCash wallets can calculate their spendable balance and the new transactions can be verified to be spending OK coins that are actually owned by the spender. The integrity and the chronological order of the block chain are enforced with cryptography.

La chaîne de blocs OK est un journal public partagé sur lequel repose tout le réseau OKCash. Toutes les transactions confirmées sont incluses dans la chaîne de blocs OK. De cette façon, les portefeuilles OKCash peuvent calculer leur solde dépensable et les nouvelles transactions afin de pouvoir vérifier la balance total qui peut être dépenser par l'utilisateur. L'intégrité et l'ordre chronologique de la chaîne de blocs sont renforcés par la cryptographie.

Transactions - private keys

Transactions - clés priver

A transaction is a transfer of value between OKCash wallets that gets included in the OK block chain. OKCash wallets keep a secret piece of data called a private key or seed, which is used to sign transactions, providing a mathematical proof that they have come from the owner of the wallet. The signature also prevents the transaction from being altered by anybody once it has been issued. All transactions are broadcasted between users and usually begin to be confirmed by the network in the following 10 seconds, through a process called staking.

Une transaction est un transfert de valeur entre les portefeuilles OKCash qui est inclus dans la chaîne de bloc OK. Les portefeuilles OKCash gardent une donnée secrète appelée clé privée ou semence, qui est utilisée pour signer des transactions, fournissant une preuve mathématique qu'ils proviennent du propriétaire du portefeuille. La signature empêche également que la transaction ne soit modifiée par quiconque une fois qu'elle a été émise. Toutes les transactions sont diffusées entre les utilisateurs et commencent généralement à être confirmées par le réseau dans les 10 secondes suivantes, grâce à un processus appelé "staking".

Processing - staking

Traitement - "staking"

Staking is a distributed consensus system that is used to confirm waiting transactions by including them in the OK block chain. It enforces a chronological order in the OK block chain, protects the neutrality of the network, and allows different computers to agree on the state of the system. To be confirmed, transactions must be packed in a block that fits very strict cryptography rules that will be verified by the network. These rules prevent previous blocks from being modified because doing so would invalidate all following blocks. Staking also creates the equivalent of a competitive lottery that prevents any individual from easily adding new blocks consecutively in the OK block chain. This way, no individuals can control what is included in the OK block chain or replace parts of the OK block chain to roll back their own spends.

Staking est un système de consensus distribué qui est utilisé pour confirmer les transactions en attente en les incluant dans la chaîne de blocs OK. Il applique un ordre chronologique dans la chaîne de blocs OK, protège la neutralité du réseau et permet à différents ordinateurs de s'entendre sur l'état du système. Pour être confirmé, les transactions doivent être regroupées dans un bloc qui respecte des règles de cryptographie très strictes qui seront vérifiées par le réseau. Ces règles empêchent la modification des blocs précédents car cela annulerait tous les blocs suivants. Staking crée également l'équivalent d'une loterie concurrentielle qui empêche toute personne d'ajouter facilement de nouveaux blocs consécutivement dans la chaîne de bloc OK. De cette façon, aucun individu ne peut contrôler ce qui est inclus dans la chaîne de bloc OK ou remplacer des parties de la chaîne de bloc OK pour annuler leurs propres dépenses.

En termes plus techniques

The weight of every address starts at 0.
When an address provides a signature, its weight increases so that after several signatures,
the weight approaches the number of coins in the address as of the last signature block:

Le poids de chaque adress commence à 0.
Quand votre adress émet une signature, sont poids augmente de façon à ce que après plusieurs signature, 
le poids s'approche du nombre de pièces dans l'adress depuis la dernière signature de bloque:

New weight = 0.9 * Old weight + 0.1 * Balance

Nouveau poid = 0.9 * vieux poid + 0.1 * Balance

If a signature is not provided by the address in a signature block, its weight decreases:

Si une signature n'est pas émis par l'adress dans un bloque de signature, sont poid diminue

New weight = 0.9 * Old weight

Nouveau poid = 0.9 * vieux poid

De cette façon, les adresses dont les propriétaires ne souhaitent pas participer à la signature ne gênent pas la capacité d'atteindre une majorité.

Si une adresse signe deux blocs en conflit, son poids est réinitialisé à 0. Ceci est pour limiter la puissance des parties prenantes malveillantes.
